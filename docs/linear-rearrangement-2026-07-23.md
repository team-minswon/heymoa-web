## 감사에서 나온 사실

**본문은 대체로 깨끗했고 낡은 참조가 첨부에 몰려 있었다.** 삭제된 계약 파일을 가리키는
첨부가 13개(APP-110의 3개 + APP-111~117의 10개)였다. 살아 있는 `agent-chat-flow.md`
첨부 2개는 그대로 두었다.

본문에서 고친 것은 둘뿐이다.

- **APP-116·117**: "서버 계획: heymoa-server `feature/app-63` 브랜치 … Task 10/11"을
  가리키고 있었다. 그 브랜치는 `dev`에 병합됐고 계획 문서의 Task 9~11은 Canceled 상태로
  web 이슈로 재생성된 것들이다. 완료된 서버 체인과 실제 계약 경로를 가리키도록 바꿨다.

요구사항 본문은 그 밖에 건드리지 않았다. 대신 구현 시 물릴 사실 둘을 참조에 덧붙였다.

- **APP-116**: 알림이 얇은 범용 구조(`type` + `resourceId`)라 응답의 `invitation`이
  초대의 **현재 상태**를 담고, 취소된 초대의 알림도 목록에 남는다
- **APP-117**: 서버가 초대 대상 조회에 이메일 정규화를 하지 않아 **대문자가 섞인 주소는
  가입한 유저인데도 404**가 될 수 있다. 초대 실패 코드 3종도 함께 적었다
