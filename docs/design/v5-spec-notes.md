# v5 SPEC 노트 전문

`mvp.pen` y52000의 SPEC 노트 4개를 옮긴 것이다. `.pen`은 git 밖이라 fresh checkout에서 캔버스를
열 수 없으므로, **수치·규칙의 정답이자 최신본은 이 파일이다.** 화면 구현(APP-153~157)은 이 문서를
근거로 한다. codex 리뷰로 정정된 항목(상단바 중지, 목록 상태 배지·필터 제거)은 이 파일에 먼저 반영됐고
캔버스 y52000 노트는 초기 버전이라 일부 뒤처질 수 있다 — 다르면 이 파일이 이긴다.

프레임 ID 표와 세대 배경은 `../design-decisions.md`를 본다.

---

## FORM SPEC (`XaNLp`) — 형태 · 타이포 · 목록 행 · 밀도

**이 노트가 하는 일:** v5 전 화면의 형태/타이포/밀도 수치를 한 곳에 못박는다. v4가 섞인 원인은
이 수치를 화면마다 각자 정했기 때문이다. 아래 값과 다르게 그려진 화면은 틀린 것이다.

**형태 스케일 5단계 — v4 계승, 바꾸지 않음**

- panel 16 / block 10 / control 8 / chip 6 / circle full
- pill(9999)은 주 CTA와 레코더 독 두 곳만. radius 0(각)은 셸에만.
- 셸(App Panel·사이드바·메인)은 r0 · 그림자 없음 · 여백 0으로 1440×900을 꽉 채운다.

**타이포 스케일 — v4 계승**

- 화면 제목 34(세리프 300) / 노트 제목 26(세리프 300) / 섹션 20 / 패널 제목 18 /
  전사 본문 15 / 목록 행 제목 15. 14 이하(탭·채팅·메타·힌트)는 건드리지 않는다.

**대문자 키커 제거 — 제품 면 전체** (drift)

- `MEETING NOTES` / `CONVERSATION` / `SUMMARY` / `PERSONAL CHATBOT` / `WORKSPACE SETTINGS` —
  제품 화면의 대문자 키커는 전부 뺀다. 탭·브레드크럼·사이드바가 이미 위치를 말하므로 중복이고
  세로 공간을 먹어 밀도를 떨어뜨린다. 세리프 300 제목은 남긴다(정체성). 랜딩·약관은 마케팅 면이라 예외.

**목록 행 정본** (drift: 코드는 높이 ~90 흰 카드 pill)

- 높이 52 · 한 줄 · r8 · 기본 배경 없음 · 행 사이 hairline · gap 14 · padding [0,12].
- 구성: 파일 아이콘 + 제목 15 + (우측) 상대 시각. **상태 배지는 두지 않는다.**
- **왜 상태 배지가 없나:** `NoteListResponse` 목록 아이템에는 `meetingStatus`도 분석 상태도 없다
  (`createdAt`·`lastRecordedAt`·`meetingStartedBy`·`recordedDurationMs`·`title`·`updatedAt`뿐).
  `meetingStatus`는 상세 `NoteResponse`에만 있다. 계약을 늘리는 안(APP-159)은 취소됐고,
  **v5는 계약 그대로 — 목록에 상태 배지를 그리지 않는다.** 회의 상태는 노트를 열면 상세에서 본다.
- **상태 의존 필터도 빼라.** "녹음 중" 필터는 목록에 `meetingStatus`가 없어 성립하지 않는다.
  필터는 `전체`와 `내가 시작`(`meetingStartedBy`로 판별)만 둔다. 캔버스 프레임에 "녹음 중" 칩이
  남아 있으면 잔재이니 구현은 이 SPEC을 따른다.

**레이아웃 산술 정정 — 노트 상세 full** (에이전트가 잡은 모순)

- 이전 브리프의 "좌 952 + 우 464"는 사이드바가 없던 v4 값이다. 사이드바를 유지하면
  255 + 952 + 464 = 1671 > 1440으로 성립하지 않는다.
- v5 정본: 사이드바 255 + 본문 1185. 본문 안에서 우측 대화 트레이 464(padding 24 →
  부양 챗 카드 416×788) + 좌측 fill(721). 트레이 padding은 24다 — 12로는 카드가 440×812가 되어
  416×788과 어긋난다. side 시트는 단일 컬럼 860, 트레이 없음.

**밀도 합격선**

- 주 콘텐츠 8개 이상 · 채팅 2왕복 이상 · 200px 넘는 빈 세로 띠 없음.
- 상태 행(y50000, 빈 상태·에러)만 예외.

## ELEVATION SPEC (`oKPxF`) — 셸은 각지고 레이어만 둥글다 (kuPpg 계승·개정)

**규칙:** 최외곽은 각지고 뷰포트를 꽉 채운다. 둥근 처리와 그림자는 실제로 뜨고 지는 안쪽 레이어에만.

| 층                                         | radius | 그림자                                  |
| ------------------------------------------ | ------ | --------------------------------------- |
| 셸(App Panel·사이드바·메인)                | 0      | 없음 · hairline으로만 구분              |
| e2 부양(챗봇 카드·플로팅 독·FAB)           | 12     | `0 2 4 #0c0a090f` + `0 10 28 #0c0a091c` |
| e3 오버레이(노트 시트·다이얼로그·드롭다운) | 12     | `0 4 8 #0c0a0914` + `0 20 56 #0c0a092b` |

그림자는 2연타다 — 짧고 진한 접지 + 길고 옅은 앰비언트. 한 겹은 회색 캔버스에서 뭉개진다.

**drift 해소**

- 개인 챗봇의 단일 티어 `0 4px 16px rgba(0,0,0,0.04)`는 흰 마케팅 페이지용이라 제품에서 안 보인다.
  → e2 2연타로 고침. 제품 화면에서 이 단일 티어를 쓰지 않는다.
- 알림 드롭다운은 불투명 배경(surface-card) + e3. 코드의 반투명은 뒤 텍스트가 비치는 결함이라 폐기.
- 설정은 다이얼로그(라우트 아님). e3 오버레이, panel 16. `rounded-[28px]`(스케일 밖 값) 폐기.

**층 판단:** 독립적으로 열리고 닫히면 다른 층 → 둥글게 + 그림자. 사이드바/메인은 항상 함께 사니 같은 셸.
카드 안의 카드(연동 카드·요약 블록·목록 행)에는 그림자 금지, hairline만. 셸에 그림자를 주면
무엇이 진짜 떠 있는지 알 수 없다.

## MOTION SPEC (`RY2I9`) — 레코더 계승 · LNplj 폐기

**레코더 3표면 — ROX9B 계승, 수치 그대로**

- A 노트 독(`recording-dock`) / B 워크스페이스 상단 pill(`workspace-toolbar`) / C 워크스페이스 밖(`global-indicator`)
- 표시 조건 배타적. 이미 구현돼 있고 유효하므로 v5가 계승한다.

**레코더 독 A — 상태별 폭 전환**

- 하나의 알약 안에서 오른쪽 내용만 교체. 왼쪽 [마이크|구분선] 고정.
- 폭 전환: `layout`(FLIP) spring, bounce 0, 0.2s. `borderRadius`를 style로 9999 못박아 둔다 —
  빼면 늘어나는 동안 알약이 타원으로 일그러진다. "인라인 스타일 정리"하다 지우기 쉬운 종류다.
- 내용 교체: `AnimatePresence popLayout`. 나가는 것 opacity→0 80ms → 폭 이동 → 100ms 지점부터
  들어오는 것 opacity 0→1 150ms. 지연 100ms가 없으면 글자가 옆으로 미끄러진다.
- 키프레임 t000(`WBWIU`) / t080(`w6xaTE`) / t250(`d3KGHq`).

**중지 버튼은 하나다** (drift)

- 노트 상세 헤더의 "중지"와 레코더 독의 붉은 원이 같은 동작이었다. 레코더 독 하나로 줄인다 —
  독은 상태별 폭 전환까지 정의된 표면이고 헤더 버튼은 중복 표기다.

**LNplj 폐기 — 사이드바 계층 push/pop을 v5는 계승하지 않는다**

- LNplj는 Cloudflare 대시보드를 100ms 간격으로 계측한 260ms 사이드바 push 명세다.
  그런데 설정은 라우트가 아니라 전체화면급 Dialog(1024×780 중앙, e3)다. 사이드바가 밀려 들어올
  구조가 앱에 없다. 없는 구조를 정당화하려 설정을 라우트로 바꾸는 것은 순서가 거꾸로다.
- → v5는 다이얼로그를 유지하고 push 스펙을 폐기한다. 형제 전환(노트 탭·설정 탭 이동)은
  콘텐츠 페인 크로스페이드(x±12, 120/180ms)로 유지한다 — 이건 대응 구조가 있다.

**접근성:** `prefers-reduced-motion`은 x 이동 전부 제거, opacity 크로스페이드만. 파형은 meter라 유지.

## CHROME SPEC (`wqUUn`) — 사이드바 · 상단바 1단 · 알림 위치

**사이드바 정본 — `v5/shell/sidebar` (sLzX8)**

- 폭 255 · 각짐 · 우측 hairline. 세 구역:
  - 상단: 워크스페이스 1줄 56 (WsBadge + 팀명 + chevron)
  - 중간: 노트 목록 (목록 행 정본 = FORM SPEC)
  - 하단: UserBar (유저 아바타 + 설정 기어, 상단 hairline)
- (drift: 코드는 상단에 유저카드 64 + 워크스페이스 스위처 56 두 겹 = 120. 유저를 하단으로 내려 120→56.)
- full 모드도 이 사이드바 255를 포함한다. 코드의 full은 워크스페이스를 덮어 내비를 잃는다 — v5는 포함.

**상단바 1단 64** (drift: 코드는 2단)

- 좌: 브레드크럼/노트 제목. 우: [노트 액션 슬롯] → 새 노트 → 벨(우측 끝, unreadCount 배지).
- 노트 액션(**회의 종료·패널 토글**)을 상단바로 올려 2단을 1단으로 합쳤다. 세로 64 회수.
- **중지는 상단바에 두지 않는다.** MOTION SPEC대로 중지 컨트롤은 레코더 독 하나뿐이다 —
  상단바 중지 + 독 = 중복(drift 7이 없애려던 것). 일부 프레임 PNG(`Ftvu9`·`AB8zp`)에 상단바
  `중지`가 남아 있으면 그건 잔재이고, 구현은 이 SPEC을 따라 독만 둔다.

**알림 위치**

- 벨은 상단바 우측 끝 유지. 위치가 틀렸던 게 아니라 주변이 비어 동떨어져 보였던 것 —
  1단 통합으로 해소. 드롭다운은 불투명 + e3(ELEVATION SPEC). 계약 `type`은 `WORKSPACE_INVITATION` 하나뿐.

**챗봇 스코프 통일** (에이전트가 발견한 갈림 조율)

- v4는 요약 탭 트레이가 개인 챗봇, 전사/정보 탭이 공유 챗봇으로 갈려 있었다. v5 정본:
  - 노트 상세 우측 트레이 = 공유 챗봇 (노트 스코프). 세 탭 모두 동일.
  - 개인 챗봇 = 워크스페이스 레벨 플로팅 (우하단 FAB → 카드). 노트에 묶이지 않는다.
- 둘은 같은 대화 UI 프리미티브(메시지 행·입력부)를 쓰고 위치·스코프만 다르다.
- **공유 챗봇은 `noteId`가 있어야 존재한다** — 노트 미선택 화면(노트 허브)에는 공유 챗봇 트레이를
  두지 않는다. 워크스페이스 레벨 대화는 개인 챗봇(플로팅)이 담당한다.

**계약 가드레일**

- 참석자: `NoteResponse`에 participants 없음 → `meetingStartedBy`만 쓴다.
- 계약에 없는 데이터를 화면이 요구하면 화면을 밀지 말고 계약 이슈를 연다.
