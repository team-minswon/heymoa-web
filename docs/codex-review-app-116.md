# APP-116 codex 리뷰 판단

로컬 `codex exec review --base dev` 4라운드. 마지막 라운드 clean(무findings)로 게이트 통과.

## 반영한 지적

- **[R1 P2] 409 인라인 + 전역 토스트 중복** — 수락/거절 mutation에 `suppressErrorToast`를
  켜고, 409(`INVITATION_NOT_PENDING`)는 인라인 Alert로만, 그 밖 실패는 `errorMessageOf`로
  직접 토스트. AGENTS 오류 경계와 일치. (반영)
- **[R2 P1] 벨이 세션 중 갱신 안 됨** — 워크스페이스 레이아웃이 계속 마운트돼 있고
  window-focus refetch가 꺼져 있어, 마운트 이후 도착한 초대가 반영되지 않음. 드롭다운을
  열 때 `refetch`. 폴링은 두지 않음(스펙 "폴링 없음" 유지). (반영)
- **[R2 P2] 좁은 뷰포트 폭** — 드롭다운 `w-[360px]`에 `max-w-[calc(100vw-1rem)]`을 더해
  375px 미만 잘림·가로 스크롤 방지. 375px 실측 확인. (반영)
- **[R2 P2 → R3 P2] 행 읽음 처리 접근성** — 처음엔 확정 행만 행-버튼으로 감쌌으나(R2),
  그러면 PENDING 행을 눌러 읽을 수 없어 스펙의 행-클릭 읽음과 충돌(R3). 최종적으로
  미읽음 **dot 자체를 "읽음으로 표시" 버튼**으로 통일 — PENDING·해결 행 모두 읽을 수 있고,
  수락/거절과 형제 버튼이라 중첩 없이 키보드·AT 접근이 된다. (반영)

## 판단

R3에서 지적한 "PENDING 행 읽음"은 a11y(버튼 중첩 금지)와 스펙(행 클릭 읽음)의 긴장이었다.
행 전체를 클릭 대상으로 만들려면 오버레이 버튼 + pointer-events 조작이 필요해 취약해진다.
dot을 읽음 버튼으로 두는 편이 더 단순하고 접근성이 명확해 그쪽을 택했다(설계 결정 갱신).
