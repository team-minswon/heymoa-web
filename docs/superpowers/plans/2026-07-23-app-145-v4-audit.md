# APP-145 v4 계약 정합성 감사 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** v4 행 여섯을 계약 기준으로 감사해 빠진 화면·상태를 메우고, 결과를 git 안 문서로 남긴다.

**Architecture:** 여섯 에이전트가 `mvp.pen`의 한 행씩 병렬로 맡는다. 실측값과 계약 발췌는 컨트롤러가 미리 준비해 넘긴다 — 에이전트는 브라우저를 띄우지 않는다. 각자 자기 파일에 기록하고 컨트롤러가 합친다.

**Spec:** `docs/superpowers/specs/2026-07-23-app-145-v4-audit-design.md`

## Global Constraints

- **`mvp.pen`은 git 밖이다.** 백업(`mvp.pen.bak-2026-07-23`)이 유일한 복구 경로다.
- **공용 `reusable` 노드 수정 금지.** 필요하면 보고만 한다.
- 새 프레임: `x = 그 행 마지막 프레임의 x + 1840`, `y`는 행 고정, 크기 **1440×900**.
- 재구성은 **자기 행 안에서, 기존 v4 토큰·컴포넌트 범위 내로**.
- 치수는 스펙의 실측표에서 파생시킨다. **추측 금지.**
- 에이전트는 `docs/design/{행}.md`에만 쓴다. `api-surface.md`·`design-decisions.md`는 컨트롤러가 쓴다.
- 리뷰 게이트는 로컬 `codex exec review --base dev` 한 번.
