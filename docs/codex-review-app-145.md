# codex 리뷰 기록 — APP-145

게이트는 로컬 `codex exec review --base dev` 하나. **`mvp.pen`은 git 밖이라 diff에 잡히지 않는다** —
codex가 본 것은 문서와 코드뿐이고, 프레임 자체는 무결성 검사(파싱·프레임 수·`reusable` 노드)와
`snapshot_layout`으로 확인했다.

| 지적 | 판단 | 근거 |
|---|---|---|
| **P2** 전역 mutation 토스트가 기존 로컬 토스트와 중복 | 고침 | **실제 통합 버그였다.** 전역 훅이 호출부 `catch`보다 먼저 돌아, 설정 저장·프로젝트 CRUD·노트 저장에서 토스트가 두 개 뜨고 문구도 서로 달랐다. 단위 테스트·lint·타입·빌드가 전부 통과했지만 이 동작은 어느 것도 덮지 않았다 |

## 어떻게 고쳤나

로컬 문구를 지우지 않고 **opt-out**했다. 프로젝트 삭제처럼 **실패 코드에 따라 문구가 갈리는 곳**이
있어 전역 문구로 대체하면 정보가 준다("프로젝트에 노트가 있어 삭제할 수 없습니다" → "요청을
처리하지 못했습니다").

플래그 이름도 바꿨다. `inlineError`는 "화면이 인라인으로 그린다"만 뜻해서 이 경우를 담지 못한다.
`suppressErrorToast`는 두 이유를 모두 덮는다 — 화면이 그리거나, 호출부가 이미 띄우거나.

중복이 재발하지 않도록 테스트로 고정했다.
